스레드와 락 관리

핵심 요약: 스레드별 TLS ID를 부여하고, RW SpinLock과 데드락 프로파일러를 연결해 서버 코어의 동시성 규칙을 관리하는 기능입니다.

문서 위치: document/architecture/thread-locking.html · 작성 기준: 2026-06-17 로컬 변경사항

요약

스레드 관리`ThreadManager`가 스레드 생성, TLS 초기화, join을 담당합니다.
RW SpinLock상위 16비트는 writer thread id, 하위 16비트는 reader count로 사용합니다.
RAII Guard`ReadLockGuard`, `WriteLockGuard`가 lock/unlock 쌍을 자동화합니다.
데드락 감지디버그 빌드에서 락 획득 순서를 그래프로 기록하고 cycle을 검사합니다.

책임

이 기능의 책임은 여러 스레드가 공유 자원에 접근할 때 쓰기/읽기 경합을 제어하고, 락 획득 순서가 꼬여 데드락으로 이어질 가능성을 개발 단계에서 감지하는 것입니다.

void ThreadManager::Launch(function<void(void)> callback)
{
    LockGuard guard(_lock);

    _threads.push_back(thread([=]()
    {
        InitTLS();
        callback();
        DestroyTLS();
    }));
}

Writer 소유권 획득

void Lock::WriteLock(const char* name)
{
    const uint32 lockThreadId = (_lockFlag.load() & WRITE_THREAD_MASK) >> 16;
    if (LThreadId == lockThreadId)
    {
        _writeCount++;
        return;
    }

    const uint32 desired = ((LThreadId << 16) & WRITE_THREAD_MASK);
    while (true)
    {
        uint32 expected = EMPTY_FLAG;
        if (_lockFlag.compare_exchange_strong(expected, desired))
        {
            _writeCount++;
            return;
        }
    }
}

Reader 카운트 증가

void Lock::ReadLock(const char* name)
{
    const uint32 lockThreadId = (_lockFlag.load() & WRITE_THREAD_MASK) >> 16;
    if (LThreadId == lockThreadId)
    {
        _lockFlag.fetch_add(1);
        return;
    }

    uint32 expected = (_lockFlag.load() & READ_COUNT_MASK);
    if (_lockFlag.compare_exchange_strong(expected, expected + 1))
        return;
}

락 순서 그래프 기록

void DeadLockProfiler::PushLock(const char* name)
{
    LockGuard guard(_lock);
    int32 lockId = 0;

    auto findIt = _nameToId.find(name);
    if (findIt == _nameToId.end())
    {
        lockId = static_cast<int32>(_nameToId.size());
        _nameToId[name] = lockId;
        _idToName[lockId] = name;
    }
    else
    {
        lockId = findIt->second;
    }

    if (_lockStack.empty() == false)
    {
        const int32 prevId = _lockStack.top();
        if (lockId != prevId)
        {
            set<int32>& history = _lockHistory[prevId];
            if (history.find(lockId) == history.end())
            {
                history.insert(lockId);
                CheckCycle();
            }
        }
    }

    _lockStack.push(lockId);
}

최근 변경

첫 커밋 기준으로 ThreadManager, RW SpinLock, lock guard, DeadLockProfiler가 함께 추가되어 스레드 실행과 락 검증을 묶는 동시성 기능 단위가 형성되었습니다.

확인 포인트

  • Reader lock CAS 조건이 writer 소유 상태를 올바르게 피하는지 확인해야 합니다.
  • WriteLock 재진입 시 `_writeCount`와 unlock 순서가 정확히 맞아야 합니다.
  • DeadLockProfiler의 `_lockStack`이 스레드별로 분리되어야 하는지 검토가 필요합니다. 현재 구현은 전역 stack 형태로 보이므로 멀티스레드 디버그 환경에서 오탐 가능성이 있습니다.
