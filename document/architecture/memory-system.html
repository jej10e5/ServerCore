메모리 관리 시스템

핵심 요약: 작은 객체 할당은 크기별 메모리 풀로 빠르게 처리하고, 큰 할당은 일반 aligned allocation으로 우회하는 서버 코어 메모리 기반 기능입니다.

문서 위치: document/architecture/memory-system.html · 작성 기준: 2026-06-17 로컬 변경사항

요약

작은 할당 최적화4096바이트 이하 요청은 크기별 `MemoryPool`을 통해 재사용됩니다.
할당 메타데이터`MemoryHeader`가 실제 데이터 앞에 붙어 해제 시 원래 할당 크기를 복원합니다.
STL 연동`StlAllocator`가 컨테이너의 내부 할당을 프로젝트 메모리 정책으로 연결합니다.
객체 생성 래퍼`xnew`, `xdelete`, `MakeShared`가 placement new와 풀 반환을 묶습니다.

책임

이 시스템의 핵심 책임은 서버에서 자주 발생하는 작은 객체 할당/해제 비용을 줄이고, 할당 경로를 한곳으로 모아 추적 가능한 형태로 만드는 것입니다. `Memory`가 정책을 결정하고, `MemoryPool`이 재사용 가능한 블록을 보관하며, `Allocator` 계층이 호출 지점을 단순화합니다.

핵심 코드

크기별 풀 테이블 구성

Memory::Memory()
{
    int32 size = 0;
    int32 tableIndex = 0;

    for (size = 32; size <= 1024; size += 32)
    {
        MemoryPool* pool = new MemoryPool(size);
        _pools.push_back(pool);

        while (tableIndex <= size)
            _poolTable[tableIndex++] = pool;
    }
}

요청 크기에 따른 할당 경로 분기

void* Memory::Allocate(int32 size)
{
    MemoryHeader* header = nullptr;
    const int32 allocSize = size + sizeof(MemoryHeader);

    if (allocSize > MAX_ALLOC_SIZE)
        header = reinterpret_cast<MemoryHeader*>(::_aligned_malloc(allocSize, SLIST_ALIGNMENT));
    else
        header = _poolTable[allocSize]->Pop();

    return MemoryHeader::AttachHeader(header, allocSize);
}

SLIST 기반 블록 재사용

MemoryHeader* MemoryPool::Pop()
{
    MemoryHeader* memory =
        static_cast<MemoryHeader*>(::InterlockedPopEntrySList(&_header));

    if (memory == nullptr)
        memory = reinterpret_cast<MemoryHeader*>(::_aligned_malloc(_allocSize, SLIST_ALIGNMENT));
    else
        _reserveCount.fetch_sub(1);

    _useCount.fetch_add(1);
    return memory;
}

template<typename Type, typename... Args>
Type* xnew(Args&&... args)
{
    Type* memory = static_cast<Type*>(PoolAllocator::Alloc(sizeof(Type)));
    new(memory)Type(forward<Args>(args)...);
    return memory;
}

최근 변경

첫 커밋 기준으로 메모리 풀, 풀 allocator, STL allocator, placement-new 기반 객체 생성 래퍼가 함께 추가되어 하나의 메모리 관리 기능 단위가 형성되었습니다.

확인 포인트

  • 4096바이트 경계에서 풀 할당과 일반 할당이 의도대로 분기되는지 확인해야 합니다.
  • `MemoryHeader::allocSize`가 해제 경로에서 항상 유효하게 유지되는지 확인해야 합니다.
  • SLIST alignment와 `_aligned_malloc`/`_aligned_free` 쌍이 깨지지 않아야 합니다.
