Project Overview

멀티스레드 IOCP 서버 구축 프로젝트

이 프로젝트는 Windows 환경에서 동작하는 멀티스레드 IOCP 서버를 단계적으로 구축하기 위한 C++ 학습용 솔루션입니다. 현재는 네트워크 기능을 붙이기 전, 서버 코어의 초기 골격과 스레드 실행 기반을 먼저 정리하는 단계입니다.

3개 솔루션 내 프로젝트
초기 단계 IOCP 이전 부트스트랩 상태
핵심 목표 스레드 기반 서버 코어 마련
현재 중심 TLS, ThreadManager, 전역 초기화

한눈에 보는 현재 상태

지금 하고 있는 일

아직 IOCP 소켓 서버를 직접 구현하는 단계는 아니고, 그 서버를 안정적으로 올릴 수 있도록 공용 코어와 멀티스레드 실행 구조를 먼저 만드는 과정입니다.

아직 없는 것

Accept, WSARecv, WSASend, IOCP 완료 통지 처리, 세션 관리, 패킷 처리 같은 실제 네트워크 로직은 아직 들어가기 전 단계입니다.

솔루션 구성

ServerCore

서버와 클라이언트 양쪽에서 공통으로 쓸 기반 코드를 담는 정적 라이브러리입니다.

ThreadManager TLS 공용 타입 공용 매크로

GameServer

현재는 실제 게임 서버라기보다, 공용 코어와 멀티스레드 동작을 테스트하는 실행 프로젝트입니다.

실행 프로젝트 스레드 테스트 부트스트랩

DummyClient

추후 테스트 클라이언트로 확장할 수 있도록 분리된 프로젝트지만, 아직은 초기 상태에 가깝습니다.

클라이언트 예정 미완성

지금까지 구현된 핵심 내용

1

공용 코어 라이브러리 분리

ServerCore를 별도 프로젝트로 분리해 공통 기능을 모아두었습니다. 이후 네트워크, 세션, 패킷, 메모리 관리 같은 기능을 여기에 확장해 넣기 좋은 구조입니다.

2

공용 타입과 동기화 별칭 정리

Types.h에서 정수형 별칭과 Atomic, Mutex, LockGuard 같은 동기화 타입 별칭을 정의했습니다.

3

TLS 기반 스레드 ID 관리

thread_local uint32 LThreadId를 사용해 각 스레드가 자기 고유 ID를 가지도록 구성했습니다. 이후 로그 추적, 스레드별 상태 관리, 스레드 로컬 자원 확장에 유리한 기반입니다.

4

ThreadManager 구현

Launch()로 작업 스레드를 생성하고, Join()으로 대기하며, 스레드 시작 시 InitTLS()를 호출해 고유 ID를 할당하는 흐름을 만들었습니다.

5

전역 코어 초기화 구조 마련

CoreGlobal이 전역 GThreadManager를 생성하고 해제합니다. 프로그램 시작 시 공용 매니저를 준비하는 패턴을 먼저 잡아둔 상태입니다.

6

서버 실행 프로젝트에서 멀티스레드 테스트

GameServer에서는 스레드 5개를 띄우고, 각 스레드가 자신의 LThreadId를 출력하도록 하여 초기 스레드 기반이 의도대로 동작하는지 확인하고 있습니다.

현재 실행 흐름

지금 서버 실행 코드는 아래와 같은 순서로 동작합니다.

  • CoreGlobal Core; 생성으로 전역 스레드 매니저 준비
  • main() 에서 작업 스레드 5개 생성
  • 각 작업 스레드는 시작하면서 InitTLS() 호출
  • 각 스레드는 자신의 LThreadId를 출력
  • 1초 간격 무한 루프로 실행 상태 유지
  • 메인 스레드는 Join()으로 스레드 종료를 대기

이 단계가 중요한 이유

바로 네트워크부터 붙이지 않음

IOCP 서버는 네트워크 코드뿐 아니라 스레드 구조, 자원 관리, 초기화 순서가 중요합니다. 그래서 먼저 기반을 안정적으로 만드는 접근을 택한 상태입니다.

확장 포인트를 먼저 확보

지금 만들어둔 구조는 이후 메모리 매니저, 네트워크 코어, 세션 매니저, 작업 큐 같은 요소를 자연스럽게 붙이기 좋은 출발점입니다.

디버깅과 운영에 유리

스레드별 식별 체계와 공용 초기화 흐름을 초기에 잡아두면, 나중에 IOCP 워커 스레드를 붙였을 때 로그 분석과 문제 추적이 훨씬 쉬워집니다.

현재 미완성 또는 확인 포인트

DummyClient

아직 실질적인 클라이언트 기능이 없습니다.

HelloWorld 호출

DummyClient.cpp의 HelloWorld() 정의는 현재 솔루션에서 확인되지 않습니다.

DestroyTLS

ThreadManager::DestroyTLS()는 아직 비어 있어, 종료 정리 로직이 추후 필요합니다.

종료 조건 부재

GameServer는 무한 루프 테스트 코드라 정상 종료 흐름은 아직 없습니다.
