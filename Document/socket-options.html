소켓 옵션과 종료

관련 커밋: 526bb81 소켓 옵션

개념 한눈에 보기

소켓 옵션은 “데이터를 어떻게 보낼까”가 아니라 “이 소켓을 어떤 정책으로 운용할까”를 정하는 장치입니다. SO_KEEPALIVE는 유휴 연결 감지, SO_LINGER는 닫을 때 남은 송신 데이터를 어떻게 다룰지, shutdown은 연결의 한 방향 또는 양방향 종료 의사를 표현합니다.

종료 관련 선택지

shutdown(SD_SEND)더 이상 보내지 않겠다는 half-close 신호
SO_LINGER onclose 시 남은 송신 데이터를 보내기 위해 일정 시간 대기
closesocket프로세스가 가진 소켓 핸들 해제
SO_KEEPALIVE조용히 죽은 연결을 TCP keep-alive로 감지

무엇을 배웠나

이 커밋은 데이터 송수신 자체보다 소켓의 동작 정책을 조절하는 옵션에 초점을 둡니다. setsockopt로 keepalive와 linger를 설정하고, shutdown과 closesocket의 역할 차이를 확인합니다.

SO_KEEPALIVE

bool enable = true;
::setsockopt(serverSocket, SOL_SOCKET, SO_KEEPALIVE, (char*)&enable, sizeof(enable));

SO_KEEPALIVE는 TCP 연결이 조용히 끊어진 경우를 감지하기 위한 옵션입니다. 상대 프로세스가 죽거나 네트워크가 사라졌는데 FIN/RST를 받지 못하면 연결이 살아 있는 것처럼 보일 수 있습니다. keepalive는 운영체제가 주기적으로 확인 패킷을 보내 끊어진 연결을 감지하게 합니다.

SO_LINGER

LINGER linger;
linger.l_onoff = 1;
linger.l_linger = 5;
::setsockopt(serverSocket, SOL_SOCKET, SO_LINGER, (char*)&linger, sizeof(linger));

SO_LINGER는 closesocket 시점에 송신 버퍼에 남은 데이터를 어떻게 처리할지 정합니다. linger를 켜고 시간을 지정하면, 닫기 전에 남은 데이터를 보내기 위해 일정 시간 기다릴 수 있습니다. 반대로 상황에 따라 빠르게 닫는 정책이 더 적절할 수도 있습니다.

shutdown과 closesocket

// Half-Close
// SD_SEND    : 더 이상 send하지 않겠다고 알림
// SD_RECEIVE : recv 방향 종료
// SD_BOTH    : 양방향 종료
::shutdown(serverSocket, SD_SEND);

::closesocket(serverSocket);

shutdown은 연결의 송신 또는 수신 방향을 먼저 닫는 프로토콜 수준의 의사 표현입니다. closesocket은 프로세스가 가진 소켓 핸들을 반환합니다. 실전 서버에서는 “더 보낼 데이터가 없다”와 “리소스를 해제한다”를 분리해서 생각해야 종료 흐름이 명확해집니다.

레벨 선택

  • SOL_SOCKET: 소켓 공통 옵션.
  • IPPROTO_IP: IPv4 계층 옵션.
  • IPPROTO_TCP: TCP 계층 옵션.
