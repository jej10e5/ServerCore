블로킹 에코 서버

관련 커밋: 41694c9 에코 클라 / 에코 서버

개념 한눈에 보기

블로킹 소켓은 호출 조건이 만족될 때까지 현재 스레드를 멈춥니다. accept는 연결 요청이 올 때까지, recv는 읽을 데이터가 도착할 때까지 기다립니다. 에코 서버는 이 단순한 성질을 이용해 “받은 만큼 다시 보내는” 네트워크 왕복 구조를 확인하기 좋은 첫 실습입니다.

에코 서버 왕복 구조

Client
send("Hello") → TCP → recv()
Server
recv()
← TCP ← send(recvBuffer)
echo

무엇을 배웠나

연결만 하던 코드에 send와 recv를 추가해 에코 서버를 구성했습니다. 클라이언트가 문자열을 보내면 서버는 받은 바이트 수만큼 다시 보내고, 클라이언트는 그 응답을 읽습니다.

서버의 echo loop

while (true)
{
    char recvBuffer[1000];

    int32 recvLen = ::recv(clientSocket, recvBuffer, sizeof(recvBuffer), 0);
    if (recvLen <= 0)
    {
        int32 errCode = ::WSAGetLastError();
        cout << "Recv ErrorCode : " << errCode << endl;
        return 0;
    }

    cout << "Recv Data! Len = " << recvLen << endl;

    int32 resultCode = ::send(clientSocket, recvBuffer, recvLen, 0);
    if (resultCode == SOCKET_ERROR)
    {
        int32 errCode = ::WSAGetLastError();
        cout << "Send ErrorCode : " << errCode << endl;
        return 0;
    }
}

클라이언트의 왕복 확인

char sendBuffer[100] = "Hello World!";
int32 resultCode = ::send(clientSocket, sendBuffer, sizeof(sendBuffer), 0);

char recvBuffer[1000];
int32 recvLen = ::recv(clientSocket, recvBuffer, sizeof(recvBuffer), 0);

cout << "Recv Data! Data = " << recvBuffer << endl;
cout << "Recv Data! Len = " << recvLen << endl;

블로킹 호출의 의미

이 코드에서는 기본 블로킹 소켓을 사용합니다. 따라서 accept는 접속할 클라이언트가 생길 때까지, recv는 읽을 데이터가 도착할 때까지 현재 스레드를 멈춥니다. 구조가 단순해 학습에 좋지만, 한 스레드에서 여러 클라이언트를 자연스럽게 처리하기 어렵습니다.

개선 포인트

  • recvLen == 0은 상대가 정상 종료한 상황이므로 오류와 구분하는 편이 좋습니다.
  • send는 요청한 길이보다 적게 보낼 수 있으므로 실전에서는 누적 전송 루프가 필요합니다.
  • 문자열 출력 시 받은 데이터가 null-terminated라는 보장이 없으므로 길이 기반으로 다루는 습관이 안전합니다.
