Winsock 기본 연결

관련 커밋: 7bf5d17 기본적인 소켓 연결 구현

개념 한눈에 보기

Winsock은 Windows에서 소켓 방식으로 네트워크 프로그램을 만들기 위한 API 계층입니다. 애플리케이션은 WSAStartup으로 Winsock DLL 사용을 시작하고, socket으로 통신 끝점을 만든 뒤 서버는 bind/listen/accept, 클라이언트는 connect로 TCP 연결을 완성합니다.

TCP 연결 준비 흐름

WSAStartup → socket → bind → listen → accept
WSAStartup → socket → connect → send/recv

무엇을 배웠나

이 단계는 TCP 서버와 클라이언트가 연결되는 최소 흐름을 만든 커밋입니다. 서버는 WinSock을 초기화한 뒤 listen socket을 만들고, bind → listen → accept 순서로 클라이언트를 기다립니다. 클라이언트는 socket → connect로 127.0.0.1:7777에 접속합니다.

서버 흐름

WSAData wsaData;
if (::WSAStartup(MAKEWORD(2, 2), &wsaData) != 0)
    return 0;

SOCKET listenSocket = ::socket(AF_INET, SOCK_STREAM, 0);

SOCKADDR_IN serverAddr = {};
serverAddr.sin_family = AF_INET;
serverAddr.sin_addr.s_addr = ::htonl(INADDR_ANY);
serverAddr.sin_port = ::htons(7777);

::bind(listenSocket, (SOCKADDR*)&serverAddr, sizeof(serverAddr));
::listen(listenSocket, 10);

SOCKADDR_IN clientAddr = {};
int32 addrLen = sizeof(clientAddr);
SOCKET clientSocket = ::accept(listenSocket, (SOCKADDR*)&clientAddr, &addrLen);

클라이언트 흐름

SOCKET clientSocket = ::socket(AF_INET, SOCK_STREAM, 0);

SOCKADDR_IN serverAddr = {};
serverAddr.sin_family = AF_INET;
::inet_pton(AF_INET, "127.0.0.1", &serverAddr.sin_addr);
serverAddr.sin_port = ::htons(7777);

if (::connect(clientSocket, (SOCKADDR*)&serverAddr, sizeof(serverAddr)) == SOCKET_ERROR)
{
    int32 errCode = ::WSAGetLastError();
    cout << "Connect ErrorCode : " << errCode << endl;
    return 0;
}

핵심 개념

  • AF_INET: IPv4 주소 체계.
  • SOCK_STREAM: TCP 스트림 소켓.
  • INADDR_ANY: 서버 머신의 모든 로컬 인터페이스에서 접속 허용.
  • htons, htonl: 호스트 바이트 오더를 네트워크 바이트 오더로 변환.
  • inet_pton: 사람이 읽는 IP 문자열을 바이너리 주소로 변환.

주의할 점

TCP 서버의 listen socket은 클라이언트와 직접 데이터를 주고받는 소켓이 아닙니다. 접속을 받아들이는 대표 소켓이고, 실제 통신은 accept가 반환한 clientSocket으로 진행됩니다.
