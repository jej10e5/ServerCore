IOCP Accept 마인드맵

중심은 IOCP다. 소켓 작업의 완료 통지를 한곳에 모으고, worker thread가 꺼내서 completion key에 담긴 객체의 Dispatch로 넘긴다.

1. 준비 단계

  • SocketUtils::Init
  • WSAStartup
  • AcceptEx 함수 포인터 로딩
  • IocpCore가 완료 큐 생성

2. 등록 대상

  • Listener는 IocpObject
  • GetHandle()로 listen socket 반환
  • Register(this)로 IOCP에 연결
  • completion key = Listener*

3. 이벤트 객체

  • IocpEvent : OVERLAPPED
  • AcceptEvent : IocpEvent
  • AcceptEvent 안에 Session*
  • 완료 후 어떤 작업인지 EventType으로 구분
IOCP 완료 통지 허브 GetQueuedCompletionStatus 완료된 작업을 worker에게 전달

4. Worker Thread

  • GIocpCore.Dispatch() 반복
  • 큐에 완료 이벤트가 올 때까지 대기
  • numOfBytes 수신
  • iocpObject, iocpEvent 수신

5. Accept 완료

  • iocpObject->Dispatch(...)
  • 실제 호출은 Listener::Dispatch
  • AcceptEvent로 캐스팅
  • ProcessAccept에서 연결 마무리

6. 안 내려올 때

  • bind 성공?
  • listen 성공?
  • AcceptEx 호출됨?
  • WSA_IO_PENDING이면 정상 대기
StartAccept문 열기
RegisterAcceptAcceptEx 걸기
IOCP Queue완료 통지 도착
Dispatch담당 객체로 전달
ProcessAccept연결 후처리

기억 포인트

Listener가 IOCP에 등록된 객체이고, AcceptEvent가 완료 시 되돌아오는 작업 표식이다. 그래서 IOCP 큐에서 꺼낸 두 포인터를 합치면 "누가 처리할지"와 "무슨 작업인지"가 모두 복원된다.
